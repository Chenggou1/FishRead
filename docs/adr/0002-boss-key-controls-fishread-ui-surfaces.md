# Boss key controls FishRead UI Surfaces

FishRead treats the boss key as a session-scoped privacy state owned by the UI Package. Entering the Boss Key Hidden State removes the FishRead UI Surface set that was active at the moment of hiding without leaving layout traces, and suspends FishRead interactions except for the restore action; restoring brings back only that saved surface set, while restored surfaces may reload their own runtime data during restore. Active overlays such as table-of-contents navigation and book management are Restorable FishRead Surfaces: they close for privacy when hidden, but capture enough interaction state for the next boss key restore to reopen the interrupted workflow where the reader left it. Hidden state is not persisted to the FishRead runtime, storage, or CLI JSON Protocol. New FishRead UI Surfaces must explicitly register with the boss key control path so future panels, widgets, and overlays cannot accidentally bypass the hidden state.
